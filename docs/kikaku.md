# 企画書

# プロジェクト名（仮）
World Sudoku（ワールド数独）

## コンセプト
数字だけではない、文字と文化で遊ぶ新しい数独体験。  
アラビア文字、ローマ文字、漢字など多様な文字を使い、  
**異なる文化を感じながら、いつもの数独を楽しむ**。


## 2. プロダクト概要

| 項目 | 内容 |
|------|------|
| プロダクト名 | World Sudoku（ワールド数独） |
| 種別 | モバイルアプリ（iOS / Android） |
| 想定技術 | Flutter + PlayFab + Firebase |
| 想定ビジネスモデル | 無料＋広告収益+広告削除 |
| リリース想定 | 2025年5月（MVP版） |

## プロダクト概要
- 数独ロジックはそのままに、**盤面の数字を多言語文字で表示**
- プレイ中に「この文字＝この数字」と自然に対応関係を学べる
- 高品質な盤面描画（画像ベース）
- デイリーパズルで継続プレイを促進
- 難易度・文字種によるスコア・ランキング要素
- 広告モデルによる収益化（広告削除課金あり）
---

## 3. 開発の背景と目的

### 背景
世界的に数独は知的パズルとして定着しているが、  
数字のみの構成では“文化的多様性”が感じづらい。  
一方で、文字や言語に興味を持つ層が増加しており、  
「ゲーム × 言語学習」という分野には伸びしろがある。

### 目的
1. **文化的な美しさと知的遊びを融合した数独体験を創出する**  
2. **多言語UIと多文字スクリプトでグローバルユーザーにリーチする**  
3. **広告＋課金で運営可能なライトパズルアプリを構築する**

---

## 4. ターゲット

| タイプ | 内容 |
|--------|------|
| パズル愛好家 | 新しいテーマ・見た目の数独を求める層 |
| カジュアル層 | 短時間で手軽にプレイしたいスマホゲーマー |
| 多言語ユーザー | 自国文字で遊びたい・文化的要素を感じたい層 |


### ペルソナ例
**アリス（28歳・翻訳家）**  
通勤中に数独で頭をリセットしたい。  
漢字やアラビア文字など異文化の文字を眺めるのが好き。

## 5. 提供価値・差別化要素

| 項目 | 内容 |
|------|------|
| **文化的多様性** | 文字そのものを使って文化を感じる仕組み |
| **安定した描画品質** | 盤面を画像ベースで生成し、文字崩れを防ぐ |
| **直感的学習効果** | 数独中に自然と「文字＝数字」を覚える |
| **グローバル対応** | 英語・日本語・アラビア語など多言語UIを標準対応 |
| **軽量・高速** | Flutterベースでスムーズな操作性を実現 |

---

## 6. コア体験（UX）

| シーン | 体験内容 |
|--------|-----------|
| 朝の通勤時 | 3分で1問。文字の形を眺めながらリフレッシュ。 |
| 夜のリラックスタイム | 高難度盤面で集中・達成感を得る。 |
| デイリーパズル | 日替わり盤面を世界の文字で遊ぶ。 |
| ランキング | 「文字種 × 難易度」で世界中とタイムを競う。 |

---

## 7. 機能構成（概要）

| カテゴリ | 機能 | 概要 |
|-----------|------|------|
| プレイ機能 | デイリーパズル |　毎日更新の盤面を自動生成（Firebase配信） |
| 〃 | 難易度選択 | 初心者〜上級者向けに4段階設定 |
| 〃 | 多言語盤面 | 各文字スクリプトを切り替え可能 |
| 競争要素 | ランキング | PlayFabを利用し、難易度・文字別にスコアを集計 |
| 収益化 | 広告表示 | AdMobによるインタースティシャル／バナー広告 
| システム | 多言語UI | 英語・日本語・アラビア語、イタリア語、フランス語、韓国語、ヒンディー語、スペイン語に切り替え対応 |
| 設定 | テーマ／音量／言語設定 | ユーザー環境に合わせた調整 |

## 8. 技術・システム方針（簡易）

| 項目 | 内容 |
|------|------|
| 開発環境 | Flutter 3.x |
| データ管理 | PlayFab（ユーザー・スコア・ランキング） |
| デイリーパズル配信 | Firebase Firestore / Hosting（JSON配信） |
| 通知 | Firebase Cloud Messaging（デイリーパズル告知） |
| 分析 | Firebase Analytics（将来的に導入予定） |
| 広告 | Google AdMob |
| UI設計 | Figmaベース、右→左対応考慮（アラビア語） |

---

## 9. 収益モデル・運用計画

### 収益モデル
- 無料ダウンロード＋広告収益（AdMob）
- 広告非表示のプレミアム課金


### 運用方針
- デイリーパズルの自動更新（Playfab）
- 新スクリプト追加やUI改善を定期リリース
- バグ修正・メンテナンスは四半期単位で実施
